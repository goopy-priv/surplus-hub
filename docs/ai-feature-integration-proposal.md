# AI Feature Integration Proposal for Surplus Hub

잉여자재 거래 플랫폼(Surplus Hub) 사용자 경험(UX) 및 참여도를 향상시키기 위한 AI 기반 기능 통합 제안서입니다. 기존 아키텍처(FastAPI 백엔드 + React/Next.js 프론트엔드)와 자재 거래, 채팅, 검색, 커뮤니티 등의 핵심 도메인을 바탕으로 구성되었습니다.

## 1. 스마트 자재 등록 보조 (Smart Material Registration)
현재 사용자는 잉여자재의 제목, 설명, 카테고리, 가격 등을 수동으로 입력해야 합니다. 공장 작업자나 관리자에게는 이 과정이 번거로울 수 있습니다.

*   **이미지 인식 및 자동 분류 (Vision AI):**
    *   **기능:** 사용자가 자재 사진(예: PVC 파이프 더미)을 업로드하면 AI가 객체를 자동으로 감지하여 **제목**을 제안(예: "100mm PVC 파이프 잔재")하고 적절한 **카테고리**를 선택합니다.
    *   **기대 효과:** 등록 단계의 번거로움과 진입 장벽을 낮춥니다.
*   **자동 설명 생성 (LLM):**
    *   **기능:** 업로드된 이미지와 몇 가지 키워드(예: "새제품, 20개")를 바탕으로 AI가 전문적이고 상세한 설명을 작성합니다(예: "미사용 새제품입니다. 수량은 20개이며 현장 직거래 희망합니다.").
*   **적정 가격 제안 (Regression / Market Data):**
    *   **기능:** 인식된 품목을 Surplus Hub 내 과거 유사 거래 데이터와 비교하여 경쟁력 있는 시장 가격(시세) 범위를 제안합니다.

## 2. 지능형 검색 및 추천 (Intelligent Search)
사용자는 필요한 자재나 공구의 정확한 기술적 명칭을 모르는 경우가 많습니다.

*   **시각적 검색 (이미지 검색):**
    *   **기능:** 부서진 부품이나 보충해야 할 자재의 사진을 찍으면 AI가 벡터 임베딩을 사용하여 플랫폼 내에서 시각적으로 유사한 매물을 찾습니다.
*   **의미론적 검색 (Semantic Search):**
    *   **기능:** 사용자가 자연어로 검색할 수 있습니다(예: "녹 안스는 튼튼한 파이프" -> "스테인리스 파이프(SUS)"). 시스템은 단순 키워드 일치가 아닌 LLM을 활용해 사용자 의도를 파악합니다.
*   **개인화 추천:**
    *   **기능:** 사용자의 과거 거래, 검색한 카테고리, 커뮤니티 활동을 분석하여 홈 화면에 관련성이 높은 자재를 추천합니다.

## 3. 스마트 채팅 어시스턴트 (Smart Chat Assistant)
B2B 또는 산업 현장에서의 거래는 흔히 협상 및 구체적인 세부 정보를 필요로 합니다.

*   **스마트 답장 추천:**
    *   **기능:** AI가 구매자의 메시지(예: "네고 가능한가요?", "어디서 거래할까요?")를 분석하고 판매자에게 3~4개의 클릭 가능한 빠른 답장 칩(예: "네, 약간은 가능합니다.", "직거래만 가능합니다.")을 제공합니다.
*   **외국인 근로자 자동 번역:**
    *   **기능:** 산업 현장에는 외국인 근로자가 많습니다. 국적이 다른 사용자 간에도 원활한 의사소통이 가능하도록 채팅방 내 실시간/양방향 자동 번역 기능을 제공합니다.
*   **사기/스팸 탐지 (안전 거래 봇):**
    *   **기능:** AI가 피싱 링크, 외부 거래 유도, 의심스러운 행동 등의 채팅 텍스트 문맥을 모니터링하여 자동으로 경고 배너를 생성합니다.

## 4. 커뮤니티 QA 및 요약 (Community QA & Summarization)
산업 내 지식 공유, 질문, 주요 주제 논의를 활성화하는 기능입니다.

*   **기술 질문 답변 봇 (Q&A Bot):**
    *   **기능:** 사용자가 커뮤니티에 기술 관련 질문을 올리면(예: "SUS304 용접할 때 가스는 뭐 쓰나요?"), (산업/용접 매뉴얼로 학습된) AI 봇이 즉각적이고 유용한 초안 답변을 제공할 수 있습니다.
*   **토론(댓글) 요약:**
    *   **기능:** 수십 개의 댓글이 달린 длин 토론 스레드의 경우, 'AI 요약' 버튼을 눌러 주요 요점과 결론(컨센서스)을 3줄 요약 형태로 빠르게 파악합니다.

## 기술적 실현 가능성 및 다음 단계 (Next Steps)
이러한 기능들은 **OpenAI (GPT-4o)** (텍스트/비전 처리), **Google Gemini 1.5 Pro**의 API를 호출하거나, 임베딩 구축을 위한 HuggingFace 오픈소스 모델 등을 활용하여 점진적으로 구현/도입할 수 있습니다.

**MVP (최소 기능 제품) 추천:**
**'스마트 자재 등록 보조(자동 설명 생성 및 이미지 검색/자동 분류)'** 기능을 최우선으로 시작할 것을 권장합니다. 이는 거래소에 공급(매물 등록)을 늘리기 위한 직접적인 병목 현상을 해소하기 때문입니다.
