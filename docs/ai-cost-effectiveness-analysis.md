# Surplus Hub AI 기능 가성비 최우선 통합 및 중국 오픈소스/API 분석 보고서

본 문서는 **"AI Feature Integration Proposal for Surplus Hub"**의 기능들을 **비용 효율성(가성비)을 최우선**으로 구현하기 위한 아키텍처 전략과, 최근 괄목할 만한가성비를 보이는 **중국 AI 오픈소스 및 API(DeepSeek, Qwen 등)** 모델에 대한 종합 분석 결과를 담고 있습니다.

---

## 1. 글로벌 vs 중국 AI API 비용 비교 (2024-2025 기준)

극단적인 가성비를 추구하기 위해서는 무조건 최고 사양의 상용 모델(GPT-4o)을 쓰는 것보다, 작업의 복잡도에 따라 모델을 다르게 가져가는 **Tiered Model Routing (계층적 모델 라우팅)** 전략이 필수적입니다. 특히 중국발 AI API의 가격 파괴가 두드러지고 있습니다.

| 모델군 | 제공사 | Input (신규) | Input (캐시) | Output | 특징 및 추천 용도 |

## 2. 가성비 극대화를 위한 기능별 구현 전략

가성비를 최우선으로 할 때, AI 구현은 외부 API 호출 비용을 최소화하고 엣지(클라이언트)/자체 호스팅 모델을 적절히 융합해야 합니다.

### A. 스마트 자재 등록 보조 (Vision AI + LLM)
가장 호출이 빈번할 수 있는 기능이므로 Vision 모델 비용 관리가 핵심입니다.

*   **1단계: Vision 처리 (이미지 인식 및 태그 추출)**
    *   **OpenAI 방식 시:** GPT-4o 등 Vision 모델은 이미지 해상도에 따라 비용이 급증합니다.
    *   **가성비 최적화 (Qwen 방식):** 알리바바의 **Qwen-VL-Max** API를 활용합니다. Qwen 계열은 시각적 이해도가 매우 뛰어나면서도 가격이 GPT-4o 대비 파격적으로 저렴합니다.
    *   **Self-Hosting 대안:** 월 등록 건수가 수천~수만 건으로 늘어날 시, GPU 서버를 임대하여 오픈소스인 **Qwen2-VL (2B 또는 7B)** 모델을 자체 호스팅하는 것이 장기적으로 훨씬 저렴합니다.
*   **2단계: 설명 생성 (Text Generation)**
    *   Vision API에서 추출한 키워드를 바탕으로 설명을 다듬는 작업은 쉬운 작업이므로 **DeepSeek-Chat (V3 API)** 또는 **GPT-4o mini**를 사용하여 비용을 최소화합니다.

### B. 지능형 검색 및 추천 (임베딩 검색)
검색 시마다 LLM을 호출하면 비용이 감당되지 않으므로, 비용이 저렴한 임베딩(Vector) 기반으로 구축해야 합니다.

*   **가성비 최적화:** 
    *   생성형 API 호출 없이 **로컬/오픈소스 임베딩 모델**을 사용합니다. 
    *   한국어 성능이 뛰어난 BAAI/bge-m3 (중국 베이징인공지능연구원 오픈소스) 모델을 FastAPI 백엔드(CPU 또는 소형 GPU)에 띄워 텍스트를 벡터화합니다. 모델 사용료는 **$0 (무료)** 입니다.
    *   **Vector DB:** 비싼 SaaS (Pinecone 등) 대신 기존에 사용 중일 가능성이 높은 **PostgreSQL + pgvector** 확장을 사용하여 인프라 비용 추가를 막습니다.

### C. 스마트 채팅 어시스턴트 & 커뮤니티 QA
실시간성이 중요하며, 대화 컨텍스트가 길어질수록 비용이 기하급수적으로 늘어납니다.

*   **스마트 자동 답장 & 요약:**
    *   **가성비 최적화:** 프롬프트 입력이 많은 작업입니다. **DeepSeek API**의 '프롬프트 캐시(Cache Hit)' 기능을 적극 활용합니다. 반복되는 시스템 프롬프트(예: "당신은 잉여자재 거래 플랫폼의 챗봇입니다...")나 긴 매뉴얼을 미리 캐싱해두면 입력 토큰 가격을 $0.07/1M 까지 떨어뜨릴 수 있습니다.
*   **스팸 필터링:**
    *   모든 메시지를 LLM으로 검사하는 것은 낭비입니다. 정규식 매칭을 거친 후, 꼭 필요한 의심 메시지만 초소형 경량 오픈소스 모델(TinyBERT 등)이나 무료에 가까운 API(DeepSeek 등)로 검사합니다.

## 3. 최종 권장 아키텍처 및 단계별 로드맵 (비용 관점)

### Phase 1: 외부 API 하이브리드 조합 (개발 속도 + 초기 비용 최소화)
직접 서버를 구축/유지보수하는 인건비가 API 호출 비용보다 비싼 초기 단계입니다. GPU 서버 구매/임대 없이 저렴한 API 조랍으로 구축합니다.
*   **Vision (이미지 인식):** Alibaba Qwen-VL API (또는 GPT-4o mini의 Vision 기능 활용)
*   **Text/QA (텍스트/요약):** DeepSeek-Chat V3 API (가장 복잡한 추론 필요시에만 R1 채택)
*   **Embedding (검색):** 서버 내 HuggingFace (bge-m3) 로컬 실행 (무료) + PostgreSQL pgvector

### Phase 2: 트래픽 증가 시 자체 호스팅 전환 (구독/API 비용 한계 도달 시)
플랫폼 활성화로 API 비용이 월 수십만 원을 초과하기 시작할 때 인프라를 전환합니다.
*   GPU가 장착된 클라우드 인스턴스(RunPod, Lambda Labs 등 저렴한 GPU 클라우드 선호) 한 대를 할당하여 vLLM을 설치합니다.
*   **Qwen2-VL** (이미지 처리용) 및 **Llama-3 (8B) / Qwen2.5 (7B)** 기반 텍스트 모델을 직접 띄워 API 호출 비용을 0원으로 만들고 서버 임대료로 고정(Fix)시킵니다.

## 📝 결론
가성비를 최우선으로 고려한다면, 기존의 "무조건 OpenAI API" 공식을 탈피해야 합니다. 특히 **DeepSeek-V3**의 캐싱 기능과 **Qwen-VL**의 저렴한 Vision 처리 능력을 적극 도입한다면, GPT-4o 단일 사용 대비 **AI 운영 비용을 최대 80~90% 이상 절감**하면서도 유사한 품질의 서비스를 사용자에게 제공할 수 있습니다.
