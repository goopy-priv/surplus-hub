# Surplus Hub AI 기능 통합 개발 계획서 (선택된 스택 기반)

본 문서는 사전에 검토된 가성비 전략 및 구체적인 모델 선정 결과를 바탕으로, 잉여자재 거래 플랫폼(Surplus Hub)에 AI 기능을 통합하기 위한 **확정된 기술 스택 및 개발 단계**를 정의합니다.

---

## 🏗️ 1. 확정된 AI 기술 스택 (Tech Stack)

비용 효율성과 작업(Task)별 최적의 성능을 고려하여 다음과 같은 멀티 모델(Multi-Model) / 하이브리드 아키텍처를 채택합니다.

### A. 스마트 자재 등록 보조 (Smart Material Registration)
가장 빈번한 호출이 예상되는 이미지 인식 및 간단한 텍스트 생성은 속도와 가성비가 매우 뛰어난 경량 API 모델로 구성합니다.
*   **Vision AI (이미지 분석 및 객체 인식):** `Gemini 2.5 Flash-Lite`
    *   **선정 이유:** 이미지 인식에 특화되어 빠르고 저렴하게 객체 카테고리와 기본 태그를 추출할 수 있습니다.
*   **Text Generation (자동 설명 생성):** `GPT-5 Nano`
    *   **선정 이유:** 가장 가벼운 최신 텍스트 생성 모델로, 추출된 태그를 바탕으로 자연스러운 문장(설명글)을 저렴한 비용으로 빠르게 합성합니다.

### B. 지능형 검색 및 추천 (Intelligent Search & Recommendation)
외부 API 호출 없이 자체 호스팅(Self-hosting)하여 쿼리당 발생하는 비용을 완전히 제거(Zero-cost per query)합니다.
*   **Embedding Model:** `BAAI/bge-m3` (오픈소스)
    *   **선정 이유:** 한국어 의미론적 검색(Semantic Search)에 탁월한 성능을 보입니다. FastAPI 백엔드의 CPU 또는 소형 GPU 환경에 직접 띄워 텍스트를 벡터화합니다.
*   **Vector Database:** `PostgreSQL + pgvector`
    *   **선정 이유:** 기존 데이터베이스(RDBMS) 아키텍처를 그대로 활용하면서도 강력한 벡터 검색(유사도 검색) 기능을 지원하여 인프라 복잡도를 낮춥니다.

### C. 스마트 채팅 어시스턴트 & 커뮤니티 QA
컨텍스트(문맥)가 길어지고 복잡한 텍스트 처리가 필요한 기능에는 가성비 최강의 중국 API를 활용합니다.
*   **Text/QA (텍스트 처리/요약/답장 추천):** `DeepSeek-Chat V3 API`
    *   **선정 이유:** 긴 문맥 캐싱(프롬프트 캐싱) 기능을 통해 반복적인 매뉴얼 참조, 스레드 요약 시 비용을 파격적으로 줄이면서도 GPT-4급 텍스트 처리 능력을 제공합니다.
*   **Complex Reasoning (복잡한 추론 - 제한적 사용):** `DeepSeek-Reasoner (R1)`
    *   **선정 이유:** 커뮤니티에서 매우 전문적이고 복잡한 엔지니어링 답변이 필요할 때만 선별적으로 호출합니다.

## 🚀 2. 개발 단계별 구현 로드맵 (Roadmap)

시스템의 안정성을 확보하고 점진적으로 가치를 제공하기 위해 3단계로 나누어 개발을 진행합니다.

### Step 1: 핵심 인프라 구축 및 검색 고도화 (인프라 & B)
기존 DB를 업그레이드하고 자체 임베딩 서버를 구축하는 기반 작업입니다.
1.  **DB 설정:** PostgreSQL에 `pgvector` 익스텐션을 설치하고 자재(Material) 테이블에 벡터 임베딩을 저장할 컬럼(`embedding_vector`)을 추가합니다.
2.  **임베딩 서버 구축:** FastAPI 백엔드 내에 `HuggingFace (sentence-transformers)` 라이브러리를 활용하여 `BAAI/bge-m3` 모델을 로드하는 엔드포인트를 구현합니다.
3.  **데이터 마이그레이션:** 기존 자재 데이터의 제목과 설명을 벡터화하여 DB에 일괄 업데이트합니다.
4.  **검색 API 개편:** 프론트엔드 검색 시 자연어를 벡터로 변환하여 DB에서 코사인 유사도(Cosine Similarity) 기반으로 쿼리하는 하이브리드 검색 API를 작성합니다.

### Step 2: 스마트 자재 등록 보조 연동 (A)
판매자(사용자)의 직접적인 편리함을 제공하는 메인 기능 구현입니다.
1.  **API Client 연동:** 백엔드에 `google-genai` (Gemini) 및 `openai` (GPT Nano) SDK를 설치합니다.
2.  **Vision 파이프라인 구축:** 프론트엔드에서 S3에 업로드된 이미지 URL(또는 Base64)을 받아 **Gemini 2.5 Flash-Lite**로 전송하고 JSON 형태(`{카테고리, 태그}`)로 응답받는 프롬프트를 작성합니다.
3.  **LLM 파이프라인 구축:** 위에서 추출된 태그와 사용자의 간단한 입력을 합쳐 **GPT-5 Nano**에 전송해 상세 설명을 완성하는 프롬프트를 구성합니다.
4.  **프론트엔드 통합:** 자재 등록 페이지 UI에 "AI 자동 작성" 버튼 및 로딩 스피너를 추가하고 응답값을 Input 폼에 자동 채워주도록 연결합니다.

### Step 3: 스마트 채팅 및 QA 봇 통합 (C)
DeepSeek API를 활용해 커뮤니티 및 채팅 기능을 한 단계 끌어올립니다.
1.  **DeepSeek API 연동:** 백엔드에 DeepSeek 계열 API 호출 클라이언트를 작성합니다. (OpenAI SDK와 호환되는 베이스 URL 활용 가능)
2.  **채팅 답장 추천:** WebSocket 채팅 서버(또는 기존 폴링 서버)에서 최근 대화 N개를 가져와 **DeepSeek V3**를 통해 "추천 답장 3개"를 생성해 클라이언트로 내려보냅니다.
3.  **커뮤니티 QA:** 커뮤니티 게시글 등록/조회 시, 특정 질문 태그가 있으면 **DeepSeek V3**(또는 필요시 R1)가 초안을 작성하여 AI 봇의 댓글로 달리도록 백그라운드 워커(Celery 등) 작업을 추가합니다.
4.  **프롬프트 캐싱 최적화:** 자주 쓰이는 시스템 프롬프트(봇의 역할 등)는 DeepSeek의 캐싱 메커니즘이 적용되도록 세션 관리를 최적화합니다.

## 🛠️ 3. 백엔드(FastAPI) 디렉토리 구조 제안

기존 코드를 건드리지 않고 독립적으로 모듈화하기 위해 다음과 같은 구조 추가를 제안합니다.

```text
/surplus-hub-api-v3/app/
├── ai/                      # AI 관련 로직 총괄 폴더
│   ├── clients/             # 외부 API 연결 및 로컬 모델 로더
│   │   ├── gemini.py        # Gemini 2.5 Flash-Lite 클라이언트
│   │   ├── gpt_nano.py      # GPT-5 Nano 클라이언트
│   │   ├── deepseek.py      # DeepSeek V3/R1 클라이언트
│   │   └── local_embed.py   # BAAI/bge-m3 모델 로딩 싱글톤 클래스
│   ├── prompts/             # 도메인별 프롬프트 분리
│   │   ├── material.py      # 자재 등록/설명 관련 프롬프트 템플릿
│   │   └── chat.py          # 답장 추천, 요약 프롬프트 템플릿
│   └── services/            # 비즈니스 로직(조합)
│       ├── registration.py  # Vision + LLM 파이프라인 조립 흐름
│       ├── search.py        # 로컬 임베딩 변환 및 pgvector 쿼리 연동
│       └── qa_bot.py        # AI 답변 및 요약 생성 로직
├── api/
│   └── routers/
│       ├── ai_assist.py     # 프론트엔드와 통신하는 앤드포인트 (AI 전용)
...
```

---
**다음 권장 액션:**
1단계인 `pgvector` 확장을 위한 데이터베이스 스키마 마이그레이션이나, 로컬 임베딩 테스트, 또는 `Gemini 2.5 Flash-Lite` 연동 파일(`gemini.py`)을 먼저 작성해 보실 수 있습니다.
