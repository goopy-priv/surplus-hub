```

| 컴포넌트 | 설명 | 상호작용 |

| 논증 구조 카드 | 주장/근거/예시를 색상별 카드로 시각화 | 읽기 전용 (자동 구성) |

| 컴포넌트 | 설명 | 상호작용 |
|---------|------|---------|
| 라이브 메모 입력 | 20자 이내 빠른 입력 + 태그 선택 (#반박/#질문/#공감) | 텍스트 입력 → 전송 |
| 메모 피드 | 팀원들의 메모가 시간순 카드 리스트로 표시 | 스크롤, 자동 갱신 |
| 교육적 반응 버튼 3종 | **[Sharp! 날카로워요]** / **[New! 새로워요]** / **[Curious 궁금해요]** | 버튼 클릭 → 반응 전송 (토글 가능) |
| 반응 카운터 | 각 메모에 받은 반응 수 표시 | 실시간 업데이트 |
| 내 메모 하이라이트 | 파란 테두리 카드 | 읽기 전용 (구분 표시) |
| 반응 알림 토스트 | "준서님이 당신의 메모에 Sharp! 반응을 보냈어요" | 자동 표시/사라짐 |

#### API 연동

| 액션 | 엔드포인트 | 비고 |
|------|-----------|------|
| 메모 작성 | `POST /api/debate/memo` | 20자 이내, 태그 선택 가능 |
| 공감 반응 | `POST /api/debate/memo/{id}/react` | Sharp/New/Curious |
| 반응 취소 | `DELETE /api/debate/memo/{id}/react` | 토글 방식 |
| 클래스 메모 피드 | `GET /api/debate/{id}/memos` | 5초 폴링 |

| 팀 협업 배지 | 팀 전체 달성 배지, 잠금/해제 상태 | 읽기 전용 |

#### 공감 리더 달성 조건

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 5명 이상의 고유한 피어(peer)로부터 반응 획득
- 반응 유형 무관 (Sharp!, New!, Curious 모두 카운트)
- 동일 학생의 중복 반응은 1명으로 카운트
- 자기 자신의 반응은 제외
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 설계 원칙

- **서열이 아닌 공감 기여도**: '논리 킹' 같은 서열 대신, 다양한 친구로부터 공감 받은 학생을 보상
- **비경쟁적 보상**: 순위가 아닌, 조건 충족 시 누구나 달성 가능
- **내성적 학생도 리더**: 메모와 공감 반응으로도 공감 리더 가능

| 섹션 | 내용 | 생성 방식 |
|------|------|----------|
| 소크라틱 스크립트 | "학생이 이런 주장을 하면, 이렇게 질문하세요" | Smart AI 자동 생성 |
| 학생 체크리스트 | 실제 이름 빈칸 체크리스트 (참여/논증 확인) | 참가자 명단 기반 |
| 공감 리더 후보 기입란 | 수업 중 메모할 빈칸 | 템플릿 자동 생성 |
| 토론 규칙 요약 | 세션 규칙, 시간 배분, 쟁점 | session_configs 기반 |

#### 연관 화면

- **A2. 토론 구조화 관리**: 쟁점/규칙/시간 데이터가 가이드에 포함
- **A5. 게이미피케이션 관리**: 공감 리더 조건이 가이드 기입란에 반영

## 4. 화면 간 관계 매핑

```
교사 관리 화면                        학생 화면
───────────                       ──────────
A1. 논증 구조 관리        ─────→   S1. 논증 구조 시각화
  (요소 토글, 난이도)               (카드 트리, AI 소크라틱)

A2. 토론 구조화 관리      ─────→   S2. 라이브 메모 & 공감 피드
  (쟁점 설정, 규칙)                (메모 입력, 교육적 반응)

A3. 메타인지 관리         ─────→   S3. 메타인지 (메모 복기 + 사고 변화)
  (성찰 질문, AI 비율)             (메모 복기, 공감 요약, 성찰)

A4. 실시간 개입 도구      ─────→   S4. 오프라인 활동 가이드
  (키워드 모니터링, 공감 리더 후보)   (학생 역할/규칙/타이머)

A5. 게이미피케이션 관리    ────→   S5. 게이미피케이션 (공감 리더 중심)
  (공감 리더 조건, 배지)            (공감 리더 배지, 개인 성장)

A6. 오프라인 가이드 생성   ────→   (교사 전용, 학생 화면 없음)
  (PDF 가이드, 자동 전송)           교사가 수업 중 PDF로 참조
```

### 데이터 흐름 방향

```
교사 설정 (A1~A6)
     │
     ├──→ A6 오프라인 가이드 자동 생성 & 전송 (수업 전)
     │
     ▼
  Backend (FastAPI)
     │
     ├──→ Smart AI Agent (분석/추천/가이드 생성)
     │         │
     │         ▼
     │    교사에게 알림 (A3, A4) + PDF 가이드 (A6)
     │
     ▼
학생 화면 (S1~S5)
     │
     ├──→ S2 라이브 메모 ──→ 키워드 분석 ──→ A4 키워드 빈도 차트
     ├──→ 교육적 반응 ──→ 공감 분석 ──→ A4 공감 리더 후보
     └──→ 학생 활동 데이터 ──→ Backend ──→ 교사 모니터링
```
